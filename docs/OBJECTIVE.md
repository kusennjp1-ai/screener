# OBJECTIVE — 目的関数（ユーザーとの約束）

このスクリーナーを「最高の聖杯」に育てるための**判断基準の優先順位**。
毎サイクルで「良くなったか？」を判定するときは、必ずこの順で見る。
上位が同点/悪化なしのときだけ、下位の改善を採用する（**上位を犠牲にした下位改善は不採用**）。

## 優先順位（不変・ユーザー合意 2026-07-23）

1. **CAGR（年率幾何平均成長率）** — 最終的に資産がどれだけ増えるか。複利の源泉。
2. **最大ドローダウン（Max Drawdown）** — 生き残り。深い谷は複利を壊し、退場リスクを生む。
3. **リスク調整後リターン** — **Sortino を主・Sharpe を従**とする。
   理由: Sharpe は上振れ（大勝ち）も「ブレ＝リスク」として罰する。我々は右テール（大勝ち）を
   **残す**戦略なので、上振れを罰する指標で自分を採点してはいけない。Sortino は**下振れだけ**を
   罰するので、この戦略の思想と一致する。両方記録し、判断は Sortino を優先。
4. **期待値（1トレードあたり／R倍）** — 1回の賭けが平均でプラスか。`expectancy_r`・`payoff_ratio`。
5. **勝率** — 最下位。ミネルヴィニ自身が「勝率は最も重要でない」と言う。
   低勝率でも payoff_ratio が高ければ +EV。勝率を上げるために大勝ちを早取りするのは**本末転倒**。

## この順位から導かれる不変ルール

- **右テール（大勝ち）を機械的に切らない**。利確固定・利益上限は入れない。
  （執行チューニング族は C71/76/80/82/85 で5連続棄却済＝実測でも大勝ちを切ると総リターン崩壊）
- **左テール（損失）は小さく**。損切りは約8%（`MAX_LOSS_PCT=0.08`）＋トレイリング。
- **勝率を目的にしない**。勝率は結果であって目標ではない。
- 数値フィッティング禁止。metric の追加・変更で見かけの改善を作らない。
- 凍結metric（FIRE±5 / GATE / golden / Band right-edge）の低下＝即revert。これは上の1〜5とは
  別の「品質の床」＝候補選定の正しさを守るガードレールで、常に維持する。

## 計測（この順位を「見える化」する場所）

- **戦術バックテスト** `backend/scripts/backtest_minervini_tactics.py` の `metrics`:
  `cagr_pct`・`max_drawdown_pct`・`sortino`・`sharpe`・`payoff_distribution`
  （`expectancy_r`・`payoff_ratio`・Rヒストグラム・右テール集中度 top5%/10%/最大1件）・`win_rate_pct`。
- **スマホPWA** の「戦略スコアカード」（`StrategyScorecardCard`）に上の5指標を優先順で表示。
  データ源: `frontend/public/static-data/strategy-scorecard.json`（バックテスト結果を転記）。
- 生成: Yahooデータはこの環境では GitHub Actions のみ→ `backtest-tactics.yml` を dispatch して取得。
