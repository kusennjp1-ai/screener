# 908トレードは出荷済みエンジンで「当時買えたか」— 再現性の直接計測

**問い（ユーザー）**: 以前の908銘柄は今の出荷済みエンジンで購入の再現が可能か。
当時そのタイミングでスクリーニングに成功し、実際に買えたか。

**手法**: `scripts/reproduce_908_buys.py`（オフライン・出荷済み `compute_vcp_footprint` 使用）。
各アイデア日 T0 の周囲 **[-15, +5] 立会日**の窓（本人は買った後にツイートすることが多いため
少し前から見る）を走査し、2つの厳しさで再現を判定:

- **detected**: `compute_vcp_footprint` が窓内で `detected=True`（＝有効なセットアップを surface できた）。
- **buy_trigger**: トレンドテンプレート健全 **かつ** 終値がVCP/ベースpivotを **≥1.4倍出来高**で上抜けた
  実際の日が窓内に存在（＝機械的に**発注できた瞬間**があった）。

## 結果（窓を持つ590アイデア）

| 層 | 定義 | 再現率 |
|---|---|---|
| **FIRE±5**（凍結・参考） | near_pivot/ready/pocket が T±5 に存在（エントリーの気配） | **91.2%** |
| **detected** | 有効VCPフットプリントが窓内に存在（セットアップ検出） | **73.1%** |
| **buy_trigger** | pivot上抜け＋出来高＋テンプレ健全（機械的に**買えた**） | **33.4%** |

年別（n / detected% / buy-trigger%）: 2020 121件 88.4%/30.6%、2018 84件 66.7%/36.9%、
2017 53件 62.3%/34.0%、2019 48件 62.5%/39.6% … （2013の21.6%が最低、2019の39.6%が最高）。
トリガー源: 一般ベースpivot 112件 / VCP検出pivot 85件。

## 結論 — 3層が設計原則をそのまま裏づける

1. **セットアップの発見はほぼできる（73–91%）**。本人の実銘柄の大半で、当時スクリーニングは
   有効なセットアップ／エントリーの気配を surface できていた。「見つける能力」は再現される。

2. **機械的に“買えた”のは約1/3（33%）**。厳格な pivot上抜け＋出来高＋テンプレ健全を満たす
   発注可能な瞬間は、彼の銘柄の約33%でしか窓内に立たない。

3. **73%→33% のギャップこそが実証された設計原則そのもの**。
   「セットアップが見える（発見・recall）」≠「機械的トリガーが発火する（執行）」。
   本人の残り2/3の買いは、pivot上抜けの機械条件では捉えられない裁量エントリー
   （早仕掛け・強さへの買い・追加建て・別ベース）だった。**発見と執行は別物**という
   C70–C72bの結論を、彼の実トレードの再現性が独立に裏づける。

## 実務的含意

- **出荷済みエンジンは「発見器」として当時から機能していた**（73–91%再現）。人間watchlistの
  表示は妥当。
- **全自動の機械執行では本人の1/3しか踏めない**。残りは裁量。よって製品は
  「機械トリガーで自動発注」ではなく「品質ランクで最良のセットアップを人間に見せる」
  （DESIGN_PRINCIPLE_SELECTION.md）が正しい翻訳。この再現性計測はその方針を追認する。

**注意**: これは凍結FIRE±5(91.2)への反証ではない。FIRE±5は「気配の近接」、buy_triggerは
「機械的発注の成立」という別の厳しさを測る、相補的な3層計測である。凍結metricは無変更
（本計測はscripts/のみ・本体サービス無変更）。
