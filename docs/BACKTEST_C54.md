# C54-C59 — ミネルヴィニ戦術ポートフォリオ・バックテスト（2026-07-10/11）

**目的**: シグナル検証(recall)ではなく**戦術そのもの**を検証する——銘柄選択・売買条件・
市場環境分析を本人のやり方どおりに執行した場合、どれだけの成績になるか。

## 手法（エンジン: `backend/scripts/backtest_minervini_tactics.py`）

- **データ**: CI（backtest-tactics.yml）でYahooから構築した6年バンドル
  （`us_universe_full.txt` 2,074銘柄・時価総額≥$2B・ETF除外＋SPY、
  `daily-price-data`リリースに`backtest-price-us-6y.json.gz`として保存）。
  パネル1,967銘柄×1,506営業日、流動性・価格・ADR≥1.5%フィルタ後1,572銘柄、
  シミュレーション **2021-07-23〜2026-07-10（約5年）**
- **銘柄選択（2ファネル）**:
  - `legacy`: 厳密8条件テンプレート＋RS percentile≥70（RS降順優先）＋
    実`VCPDetector` or 公開基準タイトベース（≥4週・深さ≤25%・直近10日≤8%）
  - `product`（C58新設・ユーザー指示）: **出荷済みBuy Signalチェックリストの機械再現**
    ——テンプレート＋RS≥70に加え、製品の`minervini_bands`ウォークフォワード履歴で
    **TPRバンド緑（strong）∧買い圧力バンド緑（buy）**を銘柄ゲート、ピボットは
    VCP箱→無ければ製品`signals._breakout_now`と同じ**直近30日もみ合い高値**、
    シグナル日に**Buy Risk緑/黄**を要求、earlyは「5営業日以内のフレッシュな
    ピボット交差（前日以下→当日超え・出来高1.5×）」限定。
    EPS Rating/Code 33はpoint-in-timeファンダ不能のため除外（製品でも推奨/ボーナス扱い）
- **買い**: armed=前日までに仕込んだピボット買い逆指値（交差当日の終値確認→翌日寄付き約定）
  ／early=ブレイク後+5%以内。**翌日寄付き・片道10bps**
- **リスク/売り/市場環境**: 1トレード1.25%リスク、ストップ=max(ベース安値,−8%)、
  ≤25%/銘柄・≤10銘柄、出荷済みトレーリングラダー＋50DMA割れ大商い売り、
  SPYレジーム（FTD込み）でエクスポージャー上限・0%時は新規買い禁止

## C58で発見・修正した検証エンジンの欠陥（結果より重要）

1. **非決定性（致命的）**: 候補銘柄をPythonのsetから作っていたため、文字列ハッシュの
   プロセス毎ランダム化で「10銘柄枠をどの銘柄が埋めるか」が実行毎に変動。
   **同一データ・同一コードで+53.9%（CI）と+24.3%（sandbox）**という乖離を確認。
   → RS降順（リーダー優先＝本人の優先則）に固定。シード変えて2回再実行し
   経済的に同一（全トレード・全指標一致）を確認。**以前の+53.9%は偶然の1試行であり
   無効**。浮動小数の総和順も固定（bitwise再現）。
2. **armedレーン死亡**: ウォッチリストを毎日上書きするため「前日仕込んだ買い逆指値が
   当日の交差で発火する」ことが論理的に不可能だった（armed判定は同日スキャンの
   c0≤pivotと同日終値>pivotの矛盾）。全実行の**エントリー100%がearly**
   （1〜2日遅れ・最大+5%高買い）になっていた。→ 交差判定を前日のarmedプランに
   対して行うよう修正（買い逆指値の本来の意味論）。
3. **product初回再現が緩すぎた**: フレッシュ交差条件を省略した近似では
   ウォッチリストが38.7→131銘柄/日に膨張し、ベース無しの伸び切った銘柄を
   買っていた（+26.4%・2022年−14.1%）。`_breakout_now`どおりの交差条件で修正。

**新しい日付が先頭**を期待する（出荷済み`vcp_footprint.py`は`iloc[::-1]`で反転）。

## 結論（ユーザーの問い「どれくらいアウトパフォームできるか」への回答・改訂）

1. **バグ修正後の戦術はSPY買い持ちを上回った**: +89.0% vs +83.6%、しかも
   maxDDほぼ半分（−13.9% vs −24.5%）・Sharpe 0.87 vs 0.80・2022年ベアを−4.0%で通過。
   「ブルで複利・ベアで生存」の設計目標をリターンでも初めて達成。
2. ただし**慎重に**: 生存バイアスは残る（絶対値は過大方向）、単一の5年窓、
   同日に3構成を測った以上「良い方を選んだ」効果への警戒が要る——
   ただし今回の改善は全て**独立に正当化されるバグ修正**（非決定性・armed死亡・
   時系列反転）であり、リターンを見てのパラメータ調整は一切していない。
   次の懐疑ステップ=**別窓（2016-2021等、10年バンドル）での追試**。
3. 製品チェックリスト再現(+45.4%)がlegacyに劣る主因は「圧力バンド緑」を
   銘柄ゲートに使うと静かなベースが構造的に弾かれること（Force Indexは
   ベース内でneutral）。**製品への示唆**: バンドは「シグナル日の確認」に使い、
   スキャン段階のフィルタには使わない方が良い。

## C59補足: VCP検出器recall調査（変更見送り）

908実トレードのオフライン再現でrecall 36.1%（CI実測~35%と一致——**以後egress不要で
計測可能**）。見逃しの81%は「深さの逐次収縮≥60%」ゲート。理論仮説
「最終収縮が最タイト」をコントロール付きで測定→ union rule で recall +2.8pp・
判別+0.8ppに留まり、**FIRE±5/golden 43件を動かす価値なしと判断し見送り**
（数値フィッティング禁止の規律）。検出器の本格改修はベース分割の設計から
（W型・ハンドル・複合ベース）——将来サイクル。
