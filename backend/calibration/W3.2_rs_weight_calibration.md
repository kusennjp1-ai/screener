# W3.2 — RS recency-weight calibration (real-data run)

Data-driven answer to the audit question flagged on `RelativeStrengthCalculator.PERIODS`:
does the 40/20/20/20 recency tilt actually predict forward outperformance, or is it
intuition at overfit risk? Answered on **real Yahoo OHLCV**, not fixtures or the synthetic
stand-in.

The authoritative run below is the **full US ex-ETF universe (≈$2B+ market cap)**; a
smaller 136-name run is retained as an appendix and corroborates the same verdict.

## How this was produced

```bash
cd backend
# 1) fetch real 5y daily OHLCV for the full US ex-ETF universe + SPY
PYTHONPATH=. python3 scripts/fetch_us_universe.py \
    --symbols-file scripts/us_universe_full.txt --out-dir realdata
# 2) walk-forward calibrate the candidate recency-weight configs vs SPY
PYTHONPATH=. python3 scripts/calibrate_rs_weights.py \
    --source csv --data-dir realdata --benchmark spy --asof-count 12
```

- **Source:** real Yahoo Finance daily OHLCV (5y), one CSV per symbol. The raw
  `realdata/` CSVs are `.gitignore`d (bulk market data); this report is the committed record.
- **Universe:** `scripts/us_universe_full.txt` — 2067 US common stocks with market cap
  ≥ $2B, ex-ETF/ETN/CEF/warrants/rights/units/preferred (NASDAQ screener API, all US
  listings). All 2067 + SPY fetched (0 failures); 89 too-short (recent IPOs, <300 rows)
  were skipped by the calibrator, leaving **1978 usable names**.
- **Method:** 12 walk-forward as-of dates; at each, rank every name by each config's
  recency-weighted relative outperformance vs SPY, flag the top 30% as "RS leaders",
  and record their forward excess return (stock − SPY) at T+1 / T+5 / T+21.

## Decision

**Recommended: `baseline_40_20_20_20` — KEEP the current default.**

Two independent findings from this run:

1. **The RS signal works.** On the full universe the flagged RS leaders beat SPY with
   strong statistical significance at *every* horizon (positive mean excess, win-rate
   >50%, t-stats +3 to +10, p≈0.00). This is the empirical validation the audit asked
   for — the top-30% RS cohort really does forward-outperform, at scale.
2. **The recency tilt is not the lever.** All five weight configs cluster tightly on every
   metric; no challenger beats the baseline on Sharpe **and** win-rate across a majority of
   horizons (the conservative rule in `calibrate_rs_weights.decide`). The recency-heavier
   configs are, if anything, marginally *worse* at T+5/T+21. Reweighting would be curve-fitting
   noise — exactly what the conservative rule is designed to reject.

### Effect on code

`RelativeStrengthCalculator.PERIODS` stays at the calibrated winner:

```python
PERIODS = {63: 0.40, 126: 0.20, 189: 0.20, 252: 0.20}  # 40/20/20/20
```

The recommended config **is** the current default, so the `periods` default needs **no
change**. The constructor remains `periods`-overridable, so any future re-run that does
surface a winner can inject it with no code edit.
